JavaScript

1. 原始类型、引用类新型有哪些？

原始（基本）数据类型有7种：

  • Number
  • Boolean
  • String
  • null
  • undefined
  • Symbol
  • BigInt

引用类型3种：

  • 对象类型 Object :普通对象 Object 、日期对象 Data
  • 数组类型 Array
  • 函数类型 Function ：数学函数 Math、其他函数对象 Function

2. 说出以下代码运行结果和原因

结果:

p1：{name: “ff”, age: 26}
p2：{name: “hh”, age: 18}

原因：
引用类型作为实参，在函数传参时传递的是内存地址值，p1将地址传递给person，person更改了p1引用地址中的内容，又给其重新分配引用地址，并将地址值传递给p2。

3. null是对象吗，为什么？

结论：
null不是对象

原因：
虽然typeof 测试null的类型输出为Object，但是这是JS的一个bug，因为JS初创时使用的是32位系统，考虑到性能，JS采用低位存储变量的类型，对象类型的二进制是以000开头的，而null的二进制是全0，则typeof输出Object的错误结果

5. 0.1+0.2 != 0.3 ，为什么？

浮点数0.1和0.2转为二进制时，是无限循环的值，计算机因为资源有限将其表示为近似值，因此出现精度损失。最后将相加的结果再转为10进制输出，结果为0.30000000000000004

6. 什么是BigInt

BigInt是第二种数字数据类型，当整数超过Number支持的范围时使用，该数据类型允许我们安全的使用大整数执行算术操作，能表示高分辨率时间戳，使用大整数ID等等，而不用调用库

7. 为什么需要BigInt

因为Number无法精确表示非常大的整数，在JS中，所有数字都是双精度64位浮点数表示，确切的说，JS中Number类型只能安全表示-(2^53-1)~(2^53-1)范围的数值，会将非常大的整数四舍五入，在任何超出此范围的整数都会失去精度：

console.log(9999 9999 9999 999) -> 1000 0000 0000 0000
console.log(9007199254740992 === 9007199254740993) ->true

Math.pow(2, 53) 竟然与 Math.pow(2, 53) + 1 相等，则安全范围是-(2^53-1)~(2^53-1)

8. 如何创建并使用BigInt?

两种方式：

  • 在数字末尾添加n就可以
  • 使用BigInt()构造函数 简单使用：

    10n + 20n;   // 30n
    10n - 20n;   //-10n
    +10n;    //TypeError
    -10n;    //-10n
    10n* 20n  //200n
    20n / 10n  //2n
    23n % 10n  //3n
    10n ** 3n  //1000n

const x= 10n;
++x;  // 11n
–x;   // 9n
console.log(x) // bigint

注意：

1) BigInt不支持一元加号运算符，这可能是某些程序可能依赖于+始终生成Number类型，或者抛出异常，另外，更改+的行为也会破坏asm.js代码。
2) 不允许BigInt和Number之间进行混合操作，因为隐式类型转换可能会丢失信息，

3) 不能将BigInt传递给Web API和JS内置方法，这些函数需要Number类型。
4) 当Boolean和BigInt相遇时，和Number类似，除了0n为false，其他值为true

5) 元素都为BigInt的数组都可以进行sort
6) BigInt可以进行位运算

9. typeof是否能正确判断类型？

对于原始类型来说，除了null都可以调用typeof显示正确的类型。

typeof 1 // ‘number’
typeof ‘1’ // ‘string’
typeof undefined // ‘undefined’
typeof true // ‘boolean’
typeof Symbol() // ‘symbol’

对于引用数据类型，除了function类型，其他都输出Object

typeof [] // ‘Object’
typeof {} //‘Object
typeof console.log //‘function’

因此采用typeof判断对象数据类型不合适，而采用instanceof会更好。因为instanceof运算符用来检测构造函数的prototype是否出现在某个实例的proto对象原型上。
